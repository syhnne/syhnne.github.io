0. 准备工作

0.1 下载watt toolkit

首先去microsoft store搜索watt toolkit，或者点击这里进入他的官网，下载并安装。

这是一个让你能连接github的工具，如果你不装它，你没法正常打开github官网。而且他是开源的，不用担心病毒之类，他还能加速一些别的东西，有兴趣可以去b站搜索这个软件了解一一下。

下载后打开，界面如下，勾选github的所有服务，点击【一键加速】。之后你可以直接右上角叉掉这个软件了，他会在后台继续运行。

之后你可以正常地进入GitHub了。注意，这个教程需要你全程把这个软件加速挂在后台，以后用git上传东西时也要先打开他，不然你连不上github。

0.2 注册github账号、新建仓库

如果你没有github账号，你需要注册一个。进入官网后，点击右上角sign up，按照他的提示输入邮箱、账号密码、用户名（注意，这里你用户名最好不要乱起，因为你博客的域名会是【用户名.github.io】如果你乱写，域名就会很奇怪。当然如果你打算另外购买域名那就当我没说）

账号注册好之后，需要新建一个仓库，用来存放你的博客代码。

点击左边的【create repository】，之后看到如下界面。

输入仓库名称时，必须是【用户名.github.io】的格式，比如我图片上这里就该输入【emily2513.github.io】当然这个方括号不用加，我这里加他是为了方便阅读（）这是硬性要求，如果你的仓库名字不是这个格式，或者和你的用户名不一样，会出现上传了博客，但输入域名进入网站看到的却是404 not found的问题。

3. 生成ssh keys

3.1 生成ssh

首先打开刚刚下载的git bash，他会弹出一个窗口，一开始上面的文字有【login】几个字，表示正在登录，一段时间后会变成这样。

注意，git bash里面不能直接ctrl+c，要想复制粘贴需要右键，或者使用他给的快捷键：ctrl+ins复制，shift+ins粘贴。

然后输入【ssh-keygen -t rsa -C “你的邮箱地址”】

他说要输入邮箱，但是不一定非得是邮箱，输点别的东西也可以。

之后按四次回车。如果他问你overwrite(y/n)，意思是问你这个文件已经存在了，是否覆盖他。输入y，允许他覆盖。

图中红线是你按回车的地方，第一行是你输入的命令。

4.1 常见报错

4.1.1 【hexo: command not found】

如果你输入hexo命令却看到这个，说明你的hexo还没安装。如果你安装了，参考这篇文档解决。

4.1.2 【FATAL (目录)not empty, please run ‘hexo init’ on an empty folder and then copy your files into it】

hexo需要在一个空的文件夹里面进行初始化（init），这里由于你既没有给hexo新建文件夹，又没有让他自己新建一个，直接输入hexo init会看到报错。

（这里的FATAL就是报错的意思，直译为“致命的，灾难性的”。这里可能出现各种不同的报错，想要知道原因，可以复制【fatal:】后面的内容粘贴到百度搜索。）

正确的做法是输入hexo init 【你要新建的文件夹名】。

4.1.3 【fatal: unable to access ‘https://github.com/hexojs/hexo-starter.git/‘: SSL certificate problem: unable to get local issuer certificate】

网络问题，多试几次就好了。实在不行你可以把后台的steam++关一下，再重新打开。

这个时候输入【cd [你新建的文件夹名称]】就能进入文件夹，然后继续操作了。

当然，有的时候即便这么修了，还是会报usage错误。

这个时候输入【npm install hexo-server –save】来更新一下hexo的服务。

然后，输入【hexo new helloworld】。这是新建一篇文章的命令，在第六章里面会讲。先别急着开写，我们还有大量bug要修（）这里先随便新建一个然后让它成功上传。

6. 开写！

6.1 发布文章的流程

博客要发布的所有文章都在【blog\source\_posts】这个路径下面。

在git中输入 hexo new post helloworld，就能新建一篇名叫helloworld的文章。

hexo new就是创建新文章的命令。post指的是你要创建的文章类型，如果你把这个词换成page，会创建一个新的【页面】，它不像post一样会被直接划到主页显示的文章范围中，而是单独列出来，以后你可以添加一些按钮来引导到这个界面。helloworld就是你要创建的文章的标题，换成任意一个词也可以，中文也行。

之后打开_posts文件夹，就能看到新创建的文章，后缀是.md，他是markdown的缩写。对这个文件进行编辑，就能修改博客内容。

修改完成之后，首先打开steam++进行加速，然后进行三件套大保健：

hexo cl && hexo g  && hexo s

就可以完成上传。

6.2 怎么写

别担心，markdown语法很简单，网上随便搜搜教程肯定是有的。hexo也有他自己的文档，会对原本的markdown写作进行一些补充：写作 | Hexo

这部分我也没研究明白，不过我有一些更简单的方法：

第一，在浏览器打开你的lofter主页，点进你要保存的文章，直接按ctrl+s，就会下载一个html文件。当然这么做的前提是你在lofter发布的文章不是很多，虽然可能有爬虫一类的方法可以批量下载，但是那个我也不会，我只能手动下载（）

之后使用这个html转md工具，把下载的html转换一下，然后用hexo new命令新建文件，把你刚才转换出来的md文件的内容全部复制，粘贴到新建的文档后面。

第二，我这篇教程是在钉钉文档写的，写完后右上角三个点里可以直接下载为md文件。

甚至不用操心图片问题了。点开这个md文件，你会发现所有的图片都给你换成了链接，复制一下链接在浏览器打开，就是你通过剪贴板加入的图片。

之后把这个md文件的内容复制粘贴到你新建的md文件后面（不要删掉他原有的东西，那些是hexo有关文章的变量，具体参考：Front-matter | Hexo）再保存，就可以发布了。

实际上钉钉文档是一个很好的免费图床，你甚至可以专门建一个文档来存图，只是我不知道他会不会偷偷把图片给压缩了。（p.s.图床就是专门用来保存图片的地方。博客上一般不把图片保存在本地，而是存在别人的服务器上，用一个图片链接来访问。如果把图片保存在本地，一是会很占地方，二是会拖慢上传速度。）
